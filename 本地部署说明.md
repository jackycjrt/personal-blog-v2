# Hugo 博客本地部署说明（Windows）

本说明仅用于在本机部署与运行当前博客项目，不设置任何快捷方式。

## 1. 环境准备

1. 安装 Hugo Extended（建议 0.120 及以上版本）。
2. 打开 PowerShell，执行：

   hugo version

3. 能看到版本号即表示 Hugo 可正常使用。
4. 当前本机已验证版本：0.159.1 Extended。

## 2. 进入项目目录

在 PowerShell 中执行：

Set-Location "e:\自己项目\个人博客搭建\版本二"

## 3. 本地启动预览

执行：

hugo server --disableFastRender

默认访问地址：

http://localhost:1313

说明：

1. 这是本机即时预览模式，修改内容后会自动刷新。
2. 关闭终端后服务停止，不会残留后台快捷启动项。

## 4. 亮暗模式说明

1. 顶部导航右侧提供亮暗切换按钮。
2. 默认模式由 config.toml 中 params.defaultTheme 控制（当前为 auto）。
3. 用户手动切换后会写入浏览器本地存储，下次访问保持上次模式。

## 5. 内容目录规范

1. 博客文章：content/posts
2. 项目页面：content/projects
3. 杂记页面：content/notes
4. 关于页面：content/about/index.md
5. 文章模板：archetypes/default.md

## 6. 主题与配置位置

1. 站点配置：config.toml
2. 主题样式：themes/anthropic-minimal/assets/css/main.css
3. 主题脚本：themes/anthropic-minimal/assets/js/main.js
4. 模板文件：themes/anthropic-minimal/layouts

## 7. 生产构建

执行：

hugo --minify

构建产物目录：

public

## 8. 常见问题

1. 页面无样式：确认 themes/anthropic-minimal/assets 下存在 css 与 js 文件。
2. 菜单不显示：确认 config.toml 中 menu.main 配置完整且 pageRef 正确。
3. 标签页空白：确认文章 front matter 中 tags 字段非空。
4. 主题未生效：确认 config.toml 中 theme = "anthropic-minimal"。
5. 命令找不到：若 winget 已安装但 `hugo` 无法识别，检查 `C:\Users\Ycjrt\AppData\Local\Microsoft\WinGet\Links\hugo.exe` 是否存在，并将该目录加入用户 PATH。

## 9. Obsidian 到 Hugo 同步工作流

已按当前需求配置以下参数：

1. Obsidian Vault 路径：C:\Users\Ycjrt\OneDrive\应用\remotely-save\yu
2. 发布源目录：C:\Users\Ycjrt\OneDrive\应用\remotely-save\yu\40_知识库\Blog
3. 图片策略：跟文章同目录（同步时保留目录结构）
4. 元信息规则：固定字段，draft 默认 true
5. 安全策略：先导出到临时目录，不会清空全站 content

执行同步（并自动构建）命令：

PowerShell -ExecutionPolicy Bypass -File .\scripts\sync-obsidian-to-hugo.ps1 -RunBuild

同步产物目录（由脚本托管）：

1. content/posts/obsidian
2. content/notes/obsidian
3. content/projects/obsidian

## 10. GitHub Pages 自动发布

已创建工作流文件：

1. .github/workflows/deploy-pages.yml

触发方式（按需求 C）：

1. 推送到 main 自动发布
2. GitHub Actions 页面手动触发 workflow_dispatch 发布

## 11. 初始化云端仓库

仓库将创建在账号 jackycjrt 下，默认仓库名：personal-blog-v2。

先登录 GitHub CLI：

gh auth login

然后执行初始化脚本：

PowerShell -ExecutionPolicy Bypass -File .\scripts\init-github-pages.ps1

脚本会自动完成：

1. 初始化本地 git 仓库（若尚未初始化）
2. 建立首次提交
3. 创建远程仓库并绑定 origin
4. 推送 main 分支

## 12. 一键发布命令

日常发布可直接执行：

PowerShell -ExecutionPolicy Bypass -File .\scripts\publish.ps1

该命令会自动执行：

1. 从 Obsidian 同步
2. Hugo 构建
3. Git 提交
4. 推送到 main（触发 GitHub Pages 自动发布）
